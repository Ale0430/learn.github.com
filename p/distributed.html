Distributed Git

Fetch, merge, pull and push between multiple remote repositories.

So you’ve heard that Git is a distributed version control system. What does that really mean and how do you take advantage of it?

One of the cool things about Git versus centralized systems is that you can have more than one ‘server’ that you can push to and pull from. In Git, this is called a ‘remote’ and you can have several of them - some of which you can write to and others you can only read from.
